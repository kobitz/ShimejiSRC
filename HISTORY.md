## Situational synthesis layer (Layers A+B+C) — built June 15 2026

*Current state: see the `SituationModel.java` bullet and the "Developing relationships" item in `CLAUDE.md`. Fusion into all reactions + the relationship journal landed June 25 2026 (see CHANGELOG "Unreleased").*

*The* north-star lever per Ko: the perception breadth was largely built, but it was **per-event and siloed** (each `fire*Reaction`/`handleScreenQuery` took one fresh snapshot of one stream). The companion-feeling comes from a *continuous, fused* situational read. Three layers:

- **(A) cheap signal sampling (no LLM)** — a shared singleton daemon `assistant/SituationModel.java` (sibling to `CpuTempMonitor`/`DriveIndexTool`, started idempotently from the Mascot ctor when assistantMode on) keeps a bounded rolling buffer (~30-60 min) sampled every few seconds: active window/process coalesced into *app-sessions*, rolling cpu/gpu/ram/audioLevel/battery averages, a digest of recent Whisper transcripts, and activity tempo (window-switch rate, idle detection, session length, time-of-day).
- **(B) periodic synthesis (1 small LLM call, ~3-5 min cadence or salience-debounced)** — buffer + previous summary → a tiny structured `Situation` (`activity`, `state` focused/multitasking/idle/winding-down, hedged `mood`, `notable` deltas, `salience` score), held in an `AtomicReference` + short history for continuity ("third late night this week").
- **(C) consumption** — `buildSystemPrompt()` + quick-reaction prompts inject `Situation.asContextBlock()` (this IS the cross-signal **fusion** — react to one fused read, not a single raw signal); **proactivity gates on `salience`, not the 45-90s clock** (timer → ceiling, salient change → trigger, silent during high-focus/low-change flow).

**Budget (mandatory):** ONE synthesis shared across all mascots (never per-mascot), slow cadence, lowest OllamaClient priority, skipped when user-facing requests queued, tiny capped output, rule-based fallback when busy — same "synthesize into small blocks, never dump" rule as drive context; runs cheaply on the VRAM-fitting e2b default. **Risks:** small-model summary noise (ship Layer A + rule-based summary first, layer LLM after), `mood` is the creepy/wrong-prone field (keep conservative or omit early), keep summaries ephemeral (not stored facts) so they don't become a new poisoning vector. Persisting the periodic summaries to a daily log bridges to developing relationships (now built — the relationship journal).

**Built so far (Layer A):** shared idempotent daemon (`ensureStarted()` from the Mascot ctor when assistantMode on) sampling foreground app-sessions (coalesced by trailing " - App" token) + cpu/gpu/ram/audio every 5s into a rolling `Situation` snapshot (state ∈ focused/multitasking/idle/winding-down/active, activity, notable delta, `salience` 0–1) held in an `AtomicReference`; routine summary logs at FINE, state transitions / salience≥0.6 at INFO; gated by `SituationModelEnabled` (default true); LLM-free → zero generation cost.

**Layer C built (June 15 2026):** a `Mascot.situationContext()` helper injects `current().asContextBlock()` into `buildSystemPrompt()` (direct replies, in `envCtx`) and the `fireSpontaneousComment` prompt (fusion — mascots react to the fused state/activity/continuity read, not just the raw window title); spontaneous comments are **salience-*triggered* (activity-aware):** when `SituationModel.isActive()` the tick fires a comment on a FRESH salient signal (edge-triggered — `salientStamp()` vs a per-mascot `lastReactedSalientStamp`, polled every `SALIENCE_CHECK_TICKS`≈2s) instead of on a clock. A salient signal is a real change (window switch +0.6; audio start/stop, +0.5 so it crosses the 0.5 threshold alone, with on/off RMS hysteresis 600/250 against flicker; state flip +0.4) OR an **ambient activity pulse** (`AMBIENT_ACTIVE_MS`=10 min) emitted during sustained NON-focused activity so a long video/listen isn't silent for the whole stretch. The 2-min global cooldown is the spacing floor; only `idle` and *silent* deep-work (`focused` AND no audio) get NO pulse — a movie classifies as `focused` too, but audio playing marks it passive so it still pulses; true silence = no sound + no input + no window change. Each salient stamp is consumed once (attempt-at-most-once → no post-cooldown burst). Logs `[Spontaneous] salience-triggered`. The legacy timer path is retained verbatim as the fallback when `SituationModelEnabled=false`. (An earlier interim used salience only as a focused-flow *veto* on the timer; replaced by true triggering.) Fully backward-compatible (no-op when `SituationModelEnabled=false`).

**Self-reference guard (June 15 2026):** the companion must never react to itself — the bubble is non-focusable and the reply dialog is undecorated/untitled (→ foreground title "(none)", already skipped), so neither pollutes app-sessions; mascot SFX DO hit the WASAPI loopback that feeds the audio signal, so `Mascot` calls `SituationModel.noteSelfAudio()` at `clip.start()` and the model freezes the audio signal for `SELF_AUDIO_MUTE_MS` (4s). (Text comments were already invisible — no focus, no audio. Salience can only *suppress*, never *trigger*, and firing still needs a real title change + 2-min cooldown, so there was no actual comment loop — this just keeps the *read* clean for Layer B.)

**Layer B built (June 15 2026):** `maybeSynthesize()` runs on the sampling loop every `SYNTH_INTERVAL_MS` (4 min), heavily gated — skips when `OllamaClient.msSinceLastDispatch() < SYNTH_QUIET_MS` (15s; never piles onto active conversation, retries in 30s), needs ≥1 app-session, honours `SituationSynthEnabled` (default true). Feeds recent app-sessions + sensors + heuristic state to the chat model (capped `SYNTH_MAX_TOKENS`=100) for a 2-line `SUMMARY:`/`MOOD:` reply, parsed into volatile `synthSummary`/`synthMood` that each 5s `Situation` snapshot then carries; `asContextBlock()` leads with the LLM narrative when present, else the rule-based facts. Graceful: any error / Ollama-down leaves the prior summary (no spam, rule-based fallback). `OllamaClient` gained a static `msSinceLastDispatch()` (shared across all clients) purely for this gate. Mood kept conservative ('neutral' when unsure) per the creepy-field risk; logs `[Situation] synthesized:` at INFO.

**Completed June 25 2026:** fusion extended to the audio/vision/peer/user-speech quick-reactions (the `situationBackground()` helper), and the periodic summaries persist to a daily log → the relationship journal (see CHANGELOG + the `SituationModel.java` / "Developing relationships" entries in `CLAUDE.md`).
